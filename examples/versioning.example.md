# Versioning examples

## Patch bump examples (`0.2.0` -> `0.2.1`)
- README clarification with no behavioral change
- typo fixes in docs
- small helper fix with no meaningful user-visible behavior change

## Minor bump examples (`0.2.0` -> `0.3.0`)
- new command module
- meaningful change to daily or weekly coaching behavior
- new integration support
- new update-check behavior
- new QA/release workflow that changes contributor/runtime expectations

## Major bump examples (`0.x.y` -> `1.0.0` or `1.x` later)
- breaking profile/config schema changes
- install/runtime redesign that breaks prior expectations
- major behavior contract changes for users or integrators
